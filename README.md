# Paper Card

把学术论文提炼为手机竖屏友好的精读长图，方便在学术圈传播。

## 工作流程

```
你提供材料 → 我搜索确认论文 → 我出初稿 → 你审核 → 定稿 → 生成 HTML → 截图出图
```

### 输入方式

- 论文链接 / PDF / DOI
- 论文标题
- 截图、文字片段、笔记（我来找到对应论文）
- 模糊描述（我来搜索候选，与你确认）

## 内容框架

**必选模块：**
1. **主标题区** — 自拟标题（引人思考、务实有洞察）+ 论文标签
2. **为什么该关心** — 背景问题与动机
3. **论文速览卡片** — 标题/作者/会议/链接

**弹性模块（按论文特点选用）：**
4. **他们是怎么做的** — 方法 + 核心截图
5. **结果怎么样** — 关键指标 + 发现
6. **局限和启发** — 讨论与延伸

## 视觉规范

| 项 | 值 |
|---|---|
| 基准宽度 | 390px（手机竖屏） |
| 底色 | `#FAFAF8` 暖白 |
| 强调色 | `#2C3E50` 深蓝灰 |
| 点缀色 | `#8B4513` 暖棕 |
| 正文字号 | 16px |
| 标题字号 | 24px，衬线 |
| 正文 | 无衬线，行高 1.7 |
| 篇幅 | ≤6 页竖屏 |

**风格：** 学术风、克制、无冗余装饰。不要标题党，不要 AI 味。

## 目录结构

```
paper-card/
├── README.md            ← 本文件
└── papers/              ← 每篇论文一个子目录
    └── paper-name/
        ├── draft.md     ← 初稿/讨论记录
        ├── final.html   ← 定稿 HTML
        └── final.png    ← 输出长图
```

## 协作方式

1. 你提供材料，我出初稿
2. 你审核内容，我按反馈调整
3. 定稿后我生成 HTML
4. 你截长图分享
